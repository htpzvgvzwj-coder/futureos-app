## 目的
记录文档变化，让 FutureOS 拥有清楚的运行历史。

## 范围
适用于 Phase 2 Bible 更新、原则变化、产品决策记录新增、路线图变化和术语更新。

## 语义化文档版本
| 版本类型 | 含义 | 示例 |
| --- |--- |--- |
| 重大版本 | 哲学、架构或治理变化。 | 增加第四个顶层系统。 |
| 次版本 | 新增原则、工作流、框架或决策记录集。 | 新增 Guardian 声誉框架。 |
| 补丁版本 | 澄清、修正、链接修复或措辞改进。 | 澄清 SGD 默认措辞。 |

## Version 2.1.1-draft(补丁）
日期：2026-07-21

在产品决策记录里新增 PDR-013(草案,尚未接受):记录真正接入第二个 Life Graph 数据 provider(真实的 SGFinDex/开放银行接入,而不仅仅是现有的手动录入数据源)到底需要什么——从技术、法律、架构三个角度。同时如实记录了当天刚建好的 provider 接口(`app/page.jsx` 的 `LIFE_GRAPH_PROVIDERS` 注册表)现在还缺什么:没有异步支持、没有数据来源标记、没有新鲜度处理、也没有同意/撤销模型。没有代码改动——这纯粹是一份设计记录,需要指定负责人并经过合规评审。

受影响文档：
- 产品决策记录(新增 PDR-013)。
- 更新日志。

## Version 2.1.0-draft
日期：2026-07-13

把 Phase 2 Bible 从原则大纲进一步深化为更接近操作系统规格的文档体系。

受影响文档：
- 与 AI 共同构建。
- 与 OCBC 共同构建。
- 关系与共同责任。
- Guardian 运行原则。
- 更新日志。

新增内容：
- Agent 阶段规格，定义输入、输出、工具、风险等级、用户同意、审计、失败回退和人工升级规则。
- 面向工程团队的 Agent 输出契约。
- 面向每个受保护目标的 Guardian Responsibility Ledger。
- Shared Goal Contract 状态和责任边界。
- Guardian Trust Event Model、错误承认和信任修复规则。
- Guardian 问责与声誉模型。
- Guardian 绩效指标和声誉状态。
- 更高自主权限的权限递进公式。
- 防止不适合银行推荐的产品冲突规则。
- 面向 OCBC 产品编排的产品适配证据要求。
- 同意违规处理协议和月度 Guardian 报告要求。

决策：
- Bible 不应只定义产品理念，也必须定义机制、状态、公式、台账、工作流、证据和失败处理。
- 中文文档必须是英文文档的语义对应版本，而不是压缩摘要。

## 必须出现的行为
- 每当文档含义改变时更新本文件。
- 架构或自主性变化必须使用产品决策记录。
- 保持中英文版本对齐。

## 禁止出现的行为
- 不要静默改变原则。
- 重大变化不要只更新一个语言版本。
- 不要把更新日志写成发布营销文案。
